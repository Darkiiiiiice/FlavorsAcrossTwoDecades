## 技术栈确认

- **Web 框架**: Axum
- **数据库**: SQLite (sqlx)
- **异步运行时**: Tokio
- **序列化**: serde + serde_json
- **错误处理**: thiserror
- **日志**: tracing + tracing-subscriber
- **配置**: config-rs
- **LLM**: async-ollama
- **API 文档**: utoipa + utoipa-swagger-ui
- **WebSocket**: axum 内置 WebSocket 支持

---

## 开发原则

1. **避免过度设计**: 保持代码简洁，不过度抽象
2. **实时持久化**: 所有状态变更直接写入数据库，不维护大聚合对象
3. **细粒度数据模型**: 使用独立表存储，避免 JSON 字段
4. **统一错误处理**: 使用统一的错误类型体系
5. **测试驱动**: 先写测试，再写实现
6. **文档同步**: 代码和文档同步更新

## 风险和注意事项

1. **sqlx 宏编译问题**: 使用离线模式或简化查询
2. **LLM 集成复杂度**: 先实现基础功能，后续优化
3. **并发安全**: 注意 SQLite 的并发限制
4. **性能测试**: 需要实际数据量测试
5. **向后兼容**: 数据库迁移需要考虑现有数据
